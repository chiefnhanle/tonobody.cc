# Implementation decisions

- The app is intentionally forgetful. It has no persistence layer of any kind — no server, no filesystem, no database, no browser storage. `/scrap` clears the page; there is nothing to save.
- "Slash is the interface." Actions are `/` commands typed into the editor rather than visible buttons. Configuration (goal-bar size, ghost prompts) appears as popups triggered by slash commands.
- The goal bar rewards momentum, not raw volume: every typed character increases `fill` (the goal is set in characters), time decays it, and deleting text never subtracts. This keeps it forgiving — you only lose ground by pausing. The fill math lives in `app/utils/bar.ts` and is unit tested in isolation.
- The bar is a draggable HUD blob (`app/components/MomentumBar.vue`), not fixed chrome — it should feel like a little object sitting on the page. Its position/dock/shape are plain in-memory state, consistent with the no-storage rule. Default placement (one-time, until first dropped elsewhere) is a horizontal bar centered along the top edge at 60% of the screen width. Picking it up (pointerdown) always pops it into a circle "marble" for the duration of the drag; dropping it within ~80px of a screen edge docks it there as a bar (horizontal along top/bottom, vertical along left/right) at 70% of that edge's screen dimension — dropping away from every edge leaves it as a floating marble. The fill is a warm gradient revealed as it rises/extends (fixed-size background image, so the color at a given position stays put as the fill grows), oriented to match the bar (left-to-right when horizontal, bottom-to-top when vertical or circular), and it gets a soft pulse glow once full. Drain speed is user-adjustable in the `/bar` popup via `DECAY_PER_SECOND * drainSpeed` (defaults to 6x). The `/bar` popup also offers three one-tap circle presets (S/M/L → `EFFORT_PRESETS` in `app/utils/bar.ts`) above the two fine-grained sliders — sliders stay for precise control, but the presets are the low-effort path for the common case. Framed as "effort" rather than just goal size: each preset moves *both* `target` and `drainSpeed` together (S: 150 chars/2x — small goal, forgiving drain; M: 300/6x — the defaults; L: 700/8x — big goal, fastest drain), since a bigger goal alone isn't more demanding unless it also has to be typed against a faster-draining bar. A preset only highlights (gradient fill) when both values exactly match it, and un-highlights as soon as either slider is dragged off.
- Ghost lines (the faint prompts on the empty page) default from `app/utils/ghosts.ts`; user-added prompts are session-only, consistent with the no-storage rule.
- Visual style is "dark, cozy, doodle": everything — including the editor's own typed content — renders in a handwritten font (Patrick Hand is both the `sans` and `mono` family, plus the explicit `font-doodle` utility). Panels use `.tv-blob` (soft asymmetric corners + a settle-in wobble on open) and `.tv-doodle-panel` (no fill at all — just a pencil-sketched outline, drawn straight onto the page rather than a card sitting on top of it). The sketched-outline effect is two offset `box-shadow`s in `--tv-ink-soft` rather than one: a bigger one cast to the bottom-right and a small counter-stroke (half the size, opposite direction) tucked at the top-left, so it reads as a hand going over the same line twice and slightly missing, not a flat drop shadow. The offset is a `--tv-doodle-offset-x/y` custom property so it scales with the element instead of being one fixed dramatic value everywhere: the base (compact chrome — bar, status pill, nav button) is small (`3px 3.5px`), and `.tv-blob.tv-doodle-panel` — the actual card-sized panels (help/bar/ghost/theme modal, slash-command dropdown) — opts into a bigger one (`6px 7px`). That compound selector was a deliberate hook: every existing usage that pairs the two classes is already a "real" card, so no new markup was needed to split them.
- Color scheme is fully swappable and pastel-first: 5 presets (3 dark — Dusk Amber, Midnight Lavender, Forest Sage; 2 light — Cream Paper, Sky Blue) defined in `app/utils/themes.ts` as a bag of `--tv-*` CSS custom properties (bg/text/muted/dim/ink/overlay/chip/three accent stops/glow). `useColorScheme()` (`app/composables/`) holds the current theme id in `useState`, and `app.vue` reactively writes the vars onto `<html>` via `useHead`'s `htmlAttrs.style` — so every component that colors itself with `var(--tv-*)` (or the `.tv-doodle-panel`/`.tv-ghost-line`/`.tv-slider`/bar gradient, which all do) repaints instantly. Picked via the `/theme` popup; like everything else, the choice is session-only and resets on reload.
- `/about` (`app/pages/about.vue`) is a real route with a concise mission statement and a link to the GitHub repo — but it's also wired up as a slash command (mirroring `/help`, `/bar`, `/ghost`, `/theme`) since "slash is the interface" is the app's whole pitch; typing `/about` in the editor navigates there via `navigateTo('/about')`.
- The writing area (`app/pages/index.vue`) is a genuinely bounded, self-scrolling box (`h-screen overflow-hidden` on the section; `min-h-0 overflow-y-auto` on the editor), not a min-height block in normal document flow. The first version only padded the top of an ever-growing page, which worked at scroll position 0 but stopped mattering the moment typing pushed the page past one screenful — the padding scrolled away with the content instead of staying put. Bounding the box means the safe-zone gap around the momentum bar (whichever edge it's docked to) holds no matter how much is typed; overflow scrolls inside the box, the page itself never grows.
- Scrollbars are themed via `scrollbar-color`/`-width` (Firefox) and `::-webkit-scrollbar*` (Chromium/Safari) applied globally in `main.css`, using `var(--tv-dim)`/`var(--tv-muted)` so they match whatever color scheme is active instead of OS chrome. Note: headless Chromium doesn't paint scrollbars into screenshots at all (verified with a minimal sanity-check page), so this one has to be eyeballed in a real browser, not via automated visual diffing.
- On touch devices (`window.matchMedia('(pointer: coarse)')`), the bar's free-drag is disabled entirely — a tap just toggles it between a circle and whatever bar dock it was last at (`app/components/MomentumBar.vue`). Free-dragging with a finger fought with page scrolling and was fiddly to land near an edge; tap-to-toggle is deliberate and reliable instead.
- The project was renamed from "Thought Vault" to **tonobody.cc** (repo moved to `github.com/chiefnhanle/tonobody.cc`). The brand name is centralized in one place: `app.vue`'s `useHead` sets a global `titleTemplate` (`"<page> — tonobody.cc"`, falling back to bare `"tonobody.cc"`), so individual pages just set a short `title` (e.g. `/about` sets `'About'`).
- Deploys as a fully static site (`pnpm run generate` → `.output/public`), not Cloudflare's Workers/Functions runtime — there's no server-side logic anywhere in the app (no API routes, no server-only data; every feature is client `useState`), so a Functions runtime would add moving parts for no benefit. `/about` is only reachable via a client-side slash command, not a real `<a>`/`<NuxtLink>`, so it's listed explicitly in `nitro.prerender.routes` (`nuxt.config.ts`) — the static crawler starts at `/` and only follows discoverable links, and would otherwise silently skip it. Verified by running the generate build and serving `.output/public` directly.
- `/capture` (`app/utils/capture.ts`) is the one deliberate exception to "nothing leaves the browser" being about *automatic* persistence, not user-initiated export: it renders the note's plain text onto a `<canvas>` and triggers a client-side download — no network call, no server, nothing kept unless the user explicitly asks. Styled as a cozy-but-watchful snapshot (soft vignette, faint grain, viewfinder corner brackets, a fixed-orange camera-style date stamp independent of the active color scheme) using the currently active theme's `--tv-bg`/`--tv-text`/`--tv-ink` (read via the `theme` from `useColorScheme()`), so the capture always matches whatever's on screen. It also draws a thin echo of the momentum bar near the top — a track in `--tv-chip` filled left-to-right with the same three-stop accent gradient as the live `MomentumBar`, sized to whatever `progress` (0–100) was at the moment `/capture` ran — so the snapshot freezes how full the goal bar was, not just the words.
